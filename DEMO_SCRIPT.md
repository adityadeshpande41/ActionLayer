## 📝 Demo Tips

### Before Recording:
1. ✅ Have sample transcript ready (realistic meeting notes)
2. ✅ Google Calendar connected with some events
3. ✅ Jira connected (or show the connection flow)
4. ✅ At least one project created
5. ✅ Clear browser cache for clean UI
6. ✅ Close unnecessary browser tabs
7. ✅ Set browser zoom to 100%

### During Recording:
- Speak clearly and at a moderate pace
- Pause briefly between sections
- Use cursor to highlight important elements
- Show loading states (they're quick but important)
- Keep energy up - this is exciting tech!

### Sample Transcript to Use:
```
Sprint Planning Meeting - March 13, 2026

Attendees: Sarah (PM), Mike (Tech Lead), Lisa (Designer), Tom (Backend)

Sarah: Let's review the user authentication feature. Mike, what's the status?

Mike: We've completed the basic login flow, but we need to decide on the session management approach. I'm recommending we use JWT tokens instead of sessions for better scalability.

Sarah: That makes sense. Let's go with JWT tokens. Mike, you'll own the implementation. What's the timeline?

Mike: I can have it done by end of next week, March 21st.

Lisa: One concern - we haven't finalized the password reset flow design yet. That could block the authentication work.

Sarah: Good catch. Lisa, can you prioritize the password reset designs? We need those by Monday.

Lisa: Yes, I'll have mockups ready by Monday morning.

Tom: There's also a risk with our email service provider. We're hitting rate limits on the free tier. If we launch authentication without upgrading, users won't receive verification emails.

Sarah: That's a high-priority risk. Tom, can you get quotes for the paid tier and send them to me by tomorrow?

Tom: Will do.

Sarah: Great. Let's also add a task to implement email queuing as a backup. Mike, can you handle that?

Mike: Sure, I'll add it to the sprint.

Sarah: Perfect. Any other blockers?

Mike: No, we're good to go.

Sarah: Excellent. Let's ship this!
```

This transcript will generate:
- 2 decisions (JWT tokens, password reset priority)
- 2 risks (email rate limits, design dependency)
- 4 action items (implement JWT, create designs, get quotes, implement queuing)
- 1 blocker (password reset designs)
